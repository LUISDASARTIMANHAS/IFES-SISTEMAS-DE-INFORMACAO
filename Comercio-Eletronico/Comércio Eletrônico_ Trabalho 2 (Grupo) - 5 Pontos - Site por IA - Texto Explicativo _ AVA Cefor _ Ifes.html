Trabalho 2 (Grupo) - 5 Pontos - Site por IA - Texto Explicativo

Condições de conclusão

Vocabulário técnico para “programar” um site via engenharia de prompt

Em ferramentas de IA, o prompt funciona como uma especificação. Por isso, ajuda dominar termos de arquitetura da informação, UI e requisitos não-funcionais.

1) Estrutura e semântica (layout)

Layout
  • Header / Cabeçalho, Topbar
  • Navbar / Menu principal, Sticky header (fixo no scroll)
  • Main (conteúdo principal)
  • Section (seções), Container / Wrapper
  • Grid / Colunas, Row
  • Sidebar (barra lateral)
  • Footer / Rodapé (com colunas)

2) Componentes de interface (UI)

UI
  • Hero section (banner principal)
  • CTA (Call to Action): botão principal/ secundário
  • Card (cartão de serviço/case)
  • Accordion (FAQ em sanfona)
  • Tabs (abas)
  • Modal / Dialog (janela)
  • Toast / Snackbar (mensagem de sucesso/erro)
  • Carousel / Slider (carrossel)
  • Form + Inputs (nome, e-mail, telefone) + Validaçã

3) Navegação e páginas

IA
  • Sitemap (mapa do site)
  • Breadcrumbs
  • Anchor links (links âncora para seções)
  • Rotas / Routes (ex.: /servicos, /contato)
  • Páginas típicas: Home, Sobre, Serviços (lista/detalhe), Cases, Blog/Post, Contato, Política de Privacidade

4) Responsividade e comportamento

Responsivo
  • Responsive design
  • Mobile-first
  • Breakpoints (ex.: 360/768/1024)
  • Above the fold (acima da dobra)
  • Hover/Focus states (estados de interação)
  • Loading states (carregando), Empty state (sem resultados)

5) Identidade visual e “design tokens”

Design
  • Paleta de cores, Tipografia
  • Spacing (espaçamentos), Ritmo vertical
  • Border radius (arredondamento), Shadow (sombra)
  • Design system / Component library
  • Buzzwords: clean, minimal, corporate, high-contrast, accessible

6) Acessibilidade (A11y)

A11y
  • Contraste AA, foco visível
  • ARIA labels
  • Semântica HTML (header, nav, main, section, footer)
  • Alt text em imagens
  • Keyboard navigation (navegável por teclado)

7) SEO e metadados

SEO
  • Title tag, Meta description
  • H1–H2–H3 (hierarquia)
  • Canonical URL
  • Open Graph
  • Sitemap.xml e Robots.txt

8) Performance e qualidade

Performance
  • Lazy loading de imagens
  • Otimização de imagens (WebP/AVIF)
  • Minify (CSS/JS), Cache
  • Core Web Vitals (LCP/CLS/INP)

9) Back-end (se houver lógica)

Back-end
  • Auth (login), Roles (admin/visitante)
  • CMS (gerenciar conteúdo)
  • API endpoints
  • Banco de dados (tabelas: posts, leads, serviços)
  • Webhooks/Integrações (e-mail marketing, CRM)

Transforme cada item acima em uma linha do prompt, e depois valide se o gerador entregou: layout global, páginas/rotas, componentes, responsividade, acessibilidade, SEO e performance.
